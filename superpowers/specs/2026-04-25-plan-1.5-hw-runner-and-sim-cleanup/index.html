    Plan 1.5 — HW Runner Cleanup and Sim-Side TODO Closure

    Date: 2026-04-25 Branch target: feature/esp32s3-plan1.5-cleanup (off feature/esp32s3-plan1-foundation) Predecessor: Plan 1 (docs/case_study_esp32s3_plan1.md, branch feature/esp32s3-plan1-foundation) Successor: Plan 2 — Boot Path + UART (M3) and Plan 3 — GPIO + SYSTIMER + Blinky (M4)

    1. Goal

    Close exit criterion 4 from Plan 1 (full 90/90 _hw/_diff variants pass on the connected ESP32-S3-Zero) and close the four documented sim-side gaps listed in docs/case_study_esp32s3_plan1.md §"Known Gaps and Acknowledged Limitations" (MOVSP register spill, ROTW privilege check, RFWU for UF8/UF12, S32E outside-vector IllegalInstruction). Every fixed behavior gets a dedicated #[hw_oracle_test] per the §7.5 methodology rule from the ESP32-S3-Zero design spec.

    The two TODO(plan2) source comments in crates/core/src/cpu/xtensa_lx7.rs (both in the MOVSP spill arm) are removed as part of the MOVSP fix. The other three sim-side gaps don't have explicit TODO(plan2) comments in code — they're documented only in the case study's Known Gaps section, and closing them means removing them from that section.

    After Plan 1.5, Plan 1 reads as 100 % closed and the simulator's Xtensa LX7 core has zero deferred behaviors. Plan 2 (boot path + UART) inherits a clean foundation.

    2. Non-goals

    • Any peripheral work (UART, GPIO, SYSTIMER, intmatrix) — that is Plan 2 / Plan 3.
    • Any boot-path work — that is Plan 2.
    • Any new system module under crates/core/src/system/ — xtensa.rs does not exist yet, and is not added by this plan. Plan 2 introduces it.
    • Any change to the ELF flashing strategy (still Strategy C: OpenOCD program command).
    • Any changes to the four pre-existing cross-compile failures (arm-hello, firmware-ci-fixture, riscv-hello, demo_blinky) — those predate Plan 1 and remain --exclude'd.
    • Any new architecture support, decoder additions, or instruction implementations beyond what's needed to close the four sim TODOs.

    3. Context

    Plan 1 closed with 62/90 _hw/_diff variants passing and the headline fibonacci_10_diff bit-exact. The remaining 28 failures cluster into five root causes documented in docs/case_study_esp32s3_plan1.md §"Live HW oracle bank — current state". All five are in the HW runner / OpenOCD interface layer; the simulator semantics for the cases that pass agree with the chip bit-for-bit.

    In addition to the runner-side failures, Plan 1 deliberately shipped four TODO(plan2) markers in the simulator core covering MOVSP register spill, RFWU for UF8/UF12, ROTW privilege check, and the IllegalInstruction path for S32E outside an exception vector. These are documented in docs/case_study_esp32s3_plan1.md §"Known Gaps and Acknowledged Limitations".

    This plan delivers both surfaces in one branch. The work is largely independent across the nine fixes, so subagent-driven development with one fix per task is the natural execution pattern.

    4. Architecture

    4.1 Three-layer split (unchanged from Plan 1)

    This plan does not change the three-layer architecture established in Plan 1:

    • Simulator (crates/core/) — the Cpu / Bus / Peripheral abstraction.
    • Hardware oracle (crates/hw-oracle/, crates/hw-oracle-macros/) — the #[hw_oracle_test] macro and its OpenOCD-backed runtime.
    • Test fixtures (fixtures/xtensa-asm/) — hand-written Xtensa asm bytecode the oracle harness flashes and runs.

    All Plan 1.5 work is internal to the first two layers.

    4.2 HW runner extension points

    The OpenOCD wrapper in crates/hw-oracle/src/openocd.rs gains three named capabilities, each behind a small public API surface:

    1. SR dispatch table. A static HashMap<&'static str, SrAccessForm> populated lazily on first use per session. SrAccessForm is one of XtensaRsr, OcdReg, or Reg. The runner probes each form once per SR by reading and writing back the same value via each command, accepting the form whose readback round-trips correctly.
    2. Windowed-register translation. New methods read_logical_ar(idx: u8) -> u32 and write_logical_ar(idx: u8, val: u32) that read the current WindowBase SR (using the SR dispatch table from #1), compute phys = ((wb << 2) + idx) & 0x3F, and route to physical AR access. The existing read_register("aN") and write_register("aN", v) calls dispatch through this layer when the name matches the regex ^a(0|1[0-5]|[1-9])$. Other register names (PC, PS, SR names) bypass.
    3. VECBASE write sequencer. A new set_vecbase(addr: u32) that wraps whatever halt-write-flush-resume sequence the empirical investigation finds. The current write_register("vecbase", ...) call sites switch to this method.

    These three capabilities are the only API additions; everything else inside openocd.rs is internal refactoring.

    4.3 Simulator extension points

    Four narrow changes to crates/core/src/cpu/xtensa_lx7.rs:

    1. MOVSP exec arm gains a real spill loop before the existing AllocaCause exception entry. The spill writes physical AR slots (WB+1)*4 .. (WB+1)*4 + 4 to [a1 - 16, a1) per the windowed ABI when WindowStart[(WB+1) & 0xF] is set, then proceeds with the existing exception entry.
    2. RFWU exec arm generalises the existing UF4 path. The number of frames to reload is N/4 where N is encoded in the RFWU instruction's immediate (4, 8, or 12). Per-frame restore loop indexed by window-rotation offset.
    3. ROTW exec arm gains a if self.ps.ring() != 0 { return self.raise_general_exception(8); } guard at the top, where 8 is PrivilegedCause per the existing inline comment at the ROTW arm (// if ps.ring() != 0 { raise PrivilegedCause (EXCCAUSE=8) }).
    4. S32E/L32E disambiguation in step() changes the EXCM-gated fall-through. Currently when EXCM is unset, the encoding is treated as S32I.N. After this fix, when EXCM is unset and the encoding pattern matches S32E/L32E specifically (not other QRST encodings), the step raises IllegalInstruction.

    4.4 Oracle test additions

    Nine new #[hw_oracle_test] entries in crates/hw-oracle/tests/oracles.rs:

    • 4 sim-side regression tests (one per sim fix above).
    • 4 replacements for the wrong-illegal-opcode tests (root cause #4): the existing tests are removed and replaced with tests using opcodes xtensa-esp32s3-elf-as confirms are actually invalid (e.g., reserved bit-pattern 0xFFFFFF if confirmed reserved by HW probe; otherwise, an opcode the assembler rejects and which the live S3-Zero raises EXCCAUSE=0 on).
    • 1 windowed-register translation regression test that pre-loads WindowBase to a non-default value (e.g., 4), writes a3 via the runner, and asserts that physical ar15 (= (4*4 + 3) & 0x3F) holds the value.

    In addition, the four sim-side fixes naturally re-enable existing _hw tests that were #[ignore]-blocked because the sim path failed; those #[ignore] attributes are removed.

    5. Components

    5.1 crates/hw-oracle/src/openocd.rs

    Addition API Lines (rough)
    SR access form enum + per-SR cache enum SrAccessForm, fn probe_sr_form(&mut self, name: &str) -> Result<SrAccessForm>, fn read_sr(&mut self, name: &str) -> Result<u32>, fn write_sr(&mut self, name: &str, val: u32) -> Result<()> ~80
    Windowed AR translation fn read_logical_ar(&mut self, idx: u8) -> Result<u32>, fn write_logical_ar(&mut self, idx: u8, val: u32) -> Result<()> ~40
    read_register / write_register dispatch update (changed signature, same name) ~30
    VECBASE sequencer fn set_vecbase(&mut self, addr: u32) -> Result<()> ~40
    Sub-word DRAM readback workaround (internal helper used by oracle tests; documented in docs/hw-oracle-runner-setup.md) ~20

    Total: roughly +200 lines, ~50 lines refactored.

    5.2 crates/core/src/cpu/xtensa_lx7.rs

    Change Site
    MOVSP spill loop within the existing MOVSP arm, before the raise_general_exception(ALLOCA_CAUSE) call
    RFWU loop generalisation within the existing RFWU arm; replace UF4-only logic with N-frame loop
    ROTW privilege guard one-line guard at top of ROTW arm
    S32E/L32E IllegalInstruction path within the EXCM-gated disambiguation in step()

    Total: roughly +150 lines, mostly the MOVSP spill loop.

    5.3 crates/hw-oracle/tests/oracles.rs

    Nine new #[hw_oracle_test] entries. Each follows the existing pattern in the file: a closure setting up registers / memory, a closure asserting end state, and the macro emitting _sim/_hw/_diff triplets.

    5.4 EXCCAUSE constants

    The existing code uses literal EXCCAUSE codes inline (e.g., raise_general_exception(0) for IllegalInstruction, raise_general_exception(8) for PrivilegedCause as commented at the ROTW arm). The implementation plan continues this convention rather than introducing constants — the inline literals are already documented by adjacent comments. If the implementer prefers constants, they may extract EXCCAUSE_ILLEGAL_INSTRUCTION = 0 and EXCCAUSE_PRIVILEGED = 8 to crates/core/src/cpu/xtensa_sr.rs as a non-functional refactor; that decision is left to the implementer and is not load-bearing.

    5.5 docs/case_study_esp32s3_plan1.md

    Append an "Addendum: Plan 1.5 closeout" section noting: - Exit criterion 4 now reads PASS — 90/90. - Both TODO(plan2) source comments in cpu/xtensa_lx7.rs removed (both were in the MOVSP path). - The four sim-side gaps in the case study's "Known Gaps and Acknowledged Limitations" section all closed — that section reduced to "Pre-existing workspace cross-compile failures" only (which predate Plan 1). - New oracle test count. - Reference to the Plan 1.5 spec and implementation plan.

    5.6 docs/hw-oracle-runner-setup.md

    Update with notes on: - Which OpenOCD command form works for each SR class (output of probe). - The windowed-register translation behaviour (developer-facing contract: "use logical aN names; the runner handles WB"). - The VECBASE write sequence the runner uses. - The sub-word DRAM readback workaround (if the empirical investigation lands on one).

    6. Data flow

    No new data flows. The HW runner's existing flow — OracleCase constructs sim and HW state, runs both to a BREAK, diffs registers + memory + selected SRs — is unchanged. The three new HW runner capabilities (SR dispatch, windowed translation, VECBASE sequencer) are pure transparent improvements to the existing register- and SR-access paths.

    7. Error handling

    The HW runner's existing Result chain is preserved. Two new failure modes:

    1. SR probe fails for all forms. probe_sr_form returns an error indicating the SR is not accessible via any known OpenOCD command. Surfaces as a clear test failure with the SR name and the probed forms tried.
    2. Windowed register index out of range. read_logical_ar / write_logical_ar panic on idx >= 16 (programmer error; the AR window is always 16 wide).

    Sim-side error handling for the four new fixes uses the existing raise_general_exception machinery; no new error types are added to SimulationError.

    8. Testing strategy

    8.1 Per-fix rhythm

    Every fix lands as its own commit with the new oracle test in the same commit. Sequence per fix:

    1. Write the oracle test for the new behaviour.
    2. Verify _sim FAILs (or PASSes if the test is for the runner only) the way the bug predicts.
    3. Verify _hw FAILs the way the bug predicts on the live board.
    4. Land the fix.
    5. Verify both _sim and _hw PASS and _diff is bit-exact.
    6. Re-run the full HW oracle bank to confirm the fix did not regress the previously-passing 62 cases.

    8.2 Sim-path regression gate

    After every commit:

    cargo test --workspace --exclude firmware --exclude firmware-ci-fixture --exclude riscv-ci-fixture

    must stay at 461+ green. Each new sim test increments the count by 1.

    8.3 HW-path validation

    The live S3-Zero is connected for this work. After each fix:

    cargo test -p labwired-hw-oracle --features hw-oracle -- --ignored --test-threads=1

    The full HW bank runs end-of-task to confirm no regression.

    8.4 Sequencing

    Tasks are largely independent. Recommended order:

    Order Fix Why this slot
    1 HW root cause #1 (SR name dispatch) Unblocks tests gated on epc1 and other SR readback. Several other tests depend on this working.
    2 HW root cause #2 (windowed-reg translation) Unblocks the largest single batch of failing tests (~10).
    3 HW root cause #3 (VECBASE sequencer) Smaller batch (3 tests) but conceptually clean.
    4 HW root cause #4 (illegal-opcode replacements) Mechanical; depends on ad-hoc probing of which opcodes are actually invalid.
    5 HW root cause #5 (sub-word DRAM readback) The murky one; gets last so easy wins are banked first. Time-box: half a day.
    6 Sim ROTW privilege check One-line; warm-up.
    7 Sim S32E IllegalInstruction Small; clear.
    8 Sim RFWU UF8/UF12 Generalises existing UF4 logic.
    9 Sim MOVSP spill Largest; gets last so RFWU's frame-walking helpers can be reused.

    9. Exit criteria

    # Criterion Verification
    1 Sim suite stays green cargo test --workspace --exclude firmware --exclude firmware-ci-fixture --exclude riscv-ci-fixture reports ≥ 470 passing (461 baseline + at least 9 new sim tests; some fixes add more than one)
    2 All HW oracle variants pass cargo test -p labwired-hw-oracle --features hw-oracle -- --ignored --test-threads=1 reports 90/90 (or higher if new tests added cleanly)
    3 Both TODO(plan2) source comments removed and four documented sim-side gaps closed grep -c "TODO(plan2)" crates/core/src/cpu/xtensa_lx7.rs returns 0; docs/case_study_esp32s3_plan1.md "Known Gaps" section retains only the pre-existing cross-compile entry
    4 Each new behaviour has a dedicated oracle test grep -c "#\[hw_oracle_test\]" crates/hw-oracle/tests/oracles.rs increases by ≥ 9 from baseline
    5 Case study addendum reflects 90/90 + closed sim TODOs docs/case_study_esp32s3_plan1.md ends with an "Addendum: Plan 1.5 closeout" section
    6 No new #[ignore] markers added grep -c "#\[ignore\]" crates/hw-oracle/tests/oracles.rs ≤ baseline
    7 Runner setup doc updated docs/hw-oracle-runner-setup.md mentions SR dispatch, windowed translation, VECBASE sequencer, sub-word DRAM workaround

    10. Risks and mitigations

    Risk Likelihood Impact Mitigation
    HW root cause #5 (sub-word DRAM readback) won't yield to the obvious fixes Medium Medium — could blow the time budget Time-box to half a day. Fallback workaround: have the test fixture's load instruction be followed by or aT, aT, a0 so the writeback is reified through a normal AR write; document the workaround in docs/hw-oracle-runner-setup.md and accept it as the chosen behaviour.
    MOVSP spill semantics diverge from real silicon in a corner case Medium Medium — could cause a _diff mismatch on a complex spill scenario Cross-validate against ESP-IDF xtensa-rt's _WindowSpill macro (MIT-licensed, allowed per design doc §7.6 reference rule). Add a multi-frame spill oracle test as part of the MOVSP fix; bit-exact diff is the verifier.
    OpenOCD SR probe sequence triggers a target halt-state issue (e.g., one of the tried command forms leaves the target in a bad state) Low Medium — could break the session for subsequent tests Probe inside a halt context that's known-safe; on probe failure for a given form, issue an explicit reset halt before trying the next form. Cache the working form so probes happen at most once per session per SR.
    Windowed-register translation introduces a subtle off-by-one in physical-slot computation Low High — every windowed test silently writes to the wrong slot The dedicated translation regression test (Section 4.4 last bullet) catches this directly. Sim and HW paths both compute the physical slot independently; the _diff check verifies they agree.
    Replacement illegal opcodes turn out to also be valid on this LX7 variant Low Low — just probe more opcodes Use xtensa-esp32s3-elf-as as the first authority: any opcode the assembler refuses to emit is a candidate. Then probe on the live board (write, single-step, expect EXCCAUSE=0) before committing to it.
